## Implementation Reference

### Stryker

- Configure Stryker for Vitest runner
- Mutate admin router and create-customer route
- Run mutation testing; review survivors
- Add or refine tests to kill survivors, or document accepted survivors

### Typical Survivors

- Boundary conditions (empty string vs null)
- Negated conditions (e.g. `if (!error)` vs `if (error)`)
- Arithmetic mutants (e.g. `+ 1` vs `- 1`)
- Block removal (code that isn't exercised by tests)
