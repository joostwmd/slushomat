## Description

Add an `admin.createOrganization` mutation to the admin router. The procedure accepts `{ name, slug, logo?, metadata?, userId }`, validates required fields, and forwards to `auth.api.createOrganization`. Because Better Auth's createOrganization requires either session headers (creates org for that user) or a `userId` in the body (server-side, no session), we must call it **without** passing the incoming request's session headers so that `userId` is used. The admin's authorization is already enforced by `adminProcedure`.

**Note:** createUser is NOT a tRPC procedure. The frontend uses `authClient.admin.createUser` (Better Auth HTTP) directly.

## Context from Discovery

### Requirements (00-requirements.md)

- **Organization body:** name (required), slug (required), logo (optional), metadata (optional), userId (required — from step 1)
- **API:** `auth.api.createOrganization({ body: { name, slug, logo?, metadata?, userId } })`
- Better Auth creates org for specified `userId` when called server-side without session headers

### Better Auth (Context7)

From `/better-auth/better-auth`:

- **Request body:** `name`, `slug`, `logo?`, `metadata?`, `userId?`, `keepCurrentActiveOrganization?`
- **userId:** Server-only. When **no** session headers are passed, org is created for the user specified by `userId`
- **For Admins:** To create org on behalf of another user, call server-side **without** passing session headers
- **Response:** `{ id, name, slug, logo, metadata, createdAt }`

### Plan (03-plan.md)

- Uses `adminProcedure`
- Validates required fields (name, slug, userId)
- Forwards to `auth.api.createOrganization`
- Map Better Auth errors (e.g. duplicate slug) to TRPCError for consistent client handling

## Implementation Reference

### Skills: `skills/api/trpc-routers.md`, `trpc-errors.md`

**Input validation (Zod):**
```ts
.input(z.object({
  name: z.string().min(1, "Name is required"),
  slug: z.string().min(1, "Slug is required"),
  logo: z.string().optional(),
  metadata: z.record(z.unknown()).optional(),
  userId: z.string().uuid("Valid user ID required"),
}))
```

**Better Auth call (no headers for userId to apply):**
```ts
// Call WITHOUT passing ctx.headers — so Better Auth uses userId from body
const result = await auth.api.createOrganization({
  body: {
    name: input.name,
    slug: input.slug,
    logo: input.logo,
    metadata: input.metadata,
    userId: input.userId,
    keepCurrentActiveOrganization: false,
  },
  // Do NOT pass headers: headers: ctx.headers
  // When no headers, Better Auth creates org for userId
});
```

**Error mapping:** Better Auth may throw on duplicate slug, invalid userId, etc. Wrap in try/catch and map to `TRPCError` with code `CONFLICT` or `BAD_REQUEST`:

```ts
try {
  const result = await auth.api.createOrganization({ body: { ... } });
  return result;
} catch (err) {
  if (err.message?.includes("slug") || err.message?.includes("duplicate")) {
    throw new TRPCError({ code: "CONFLICT", message: "Slug already exists" });
  }
  if (err.message?.includes("user") || err.message?.includes("not found")) {
    throw new TRPCError({ code: "BAD_REQUEST", message: "User not found" });
  }
  throw new TRPCError({
    code: "INTERNAL_SERVER_ERROR",
    message: "Failed to create organization",
    cause: err,
  });
}
```
