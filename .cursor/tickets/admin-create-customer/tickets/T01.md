## Context from Discovery

### Requirements (00-requirements.md)

- **createUser API:** `authClient.admin.createUser({ email, password, name, role: "user", data? })` — Better Auth admin plugin HTTP
- **createOrganization API:** `auth.api.createOrganization({ body: { name, slug, logo?, metadata?, userId } })` — called server-side via tRPC (T02)
- Step 1 must complete before step 2 (userId required)
- No pre-check for duplicate email/slug; rely on API errors

### Test Spec (02-test-spec.md)

- **Unit tests:** Client-side required-field validation (UT-01–UT-04), payload validation if schema exists (UT-05, UT-06)
- **Integration tests:** Create admin session via Better Auth testUtils + PGlite; use unique emails/slugs per test (`test-${Date.now()}@example.com`)
- **APIs under test:** createUser via HTTP endpoint; createOrganization via tRPC procedure (hits `auth.api.createOrganization` internally)
- **Test setup:** Better Auth testUtils with PGlite in-memory Postgres; admin user with `role: "admin"`

## Implementation Reference

### Skills: `skills/testing/`

**From `_index.md`:**
- Monorepo layout: `tests/_fixtures/`, `tests/_utils/`, `tests/api/<feature>/`, `tests/web/<feature>/`
- Shared rules: Import from path aliases; never `vi.mock` TanStack Query, tRPC, or Drizzle; query by role/label/text; use `findBy*` or `waitFor` for async; one PGlite per file in `beforeAll`

**From `trpc.md`:**
- `createTestCaller({ db, userId })` — injects PGlite db and optional auth session
- `dbTest('procedure rejects duplicate', async ({ db, createUser }) => { ... })`
- Auth-gated: `createTestCaller({ db })` for no session; `createTestCaller({ db, userId: adminUser.id })` for admin

**From `db-infra.md`:** For dbSafe/withRetry — not directly relevant; createOrganization tests use tRPC caller.

**From `db-queries.md`:** `dbTest` fixture wraps each test in transaction rollback; seed with `createUser` helper.

### createUser Integration Tests

createUser is **not** a tRPC procedure. Integration tests must hit the Better Auth HTTP route `/api/auth/*` (admin createUser endpoint). Use fetch or a test HTTP client with admin session cookies from Better Auth testUtils.

### createOrganization Integration Tests

Once T02 exists, use tRPC caller:

```ts
const caller = await createTestCaller({ db, userId: adminUser.id });
const result = await caller.admin.createOrganization({
  name: "Test Org",
  slug: "test-org",
  userId: newUser.id,
});
expect(result).toBeDefined();
expect(result.id).toBeDefined();
```

### Validation Unit Tests (UT-01–UT-04) — React Testing Library

Use **React Testing Library** (`@testing-library/react`, `@testing-library/user-event`) to render the create-customer wizard and test validation:

- Render the form (mock tRPC and authClient or use providers with no-op mocks)
- Use `userEvent` to fill fields (or leave empty) and click submit
- Assert that error messages appear (e.g. `screen.getByRole('alert')`, `getByText(/required/i)`)
- UT-01: Submit step 1 with all empty
- UT-02: Submit step 1 with only email
- UT-03: Submit step 2 with name and slug empty
- UT-04: Submit step 2 with only name

Place tests in `apps/admin-frontend/src/routes/_admin/create-customer.test.tsx` (alongside the component).
