# T01 — Add One-Time Token Plugin to Better Auth

**Task ID:** T01  
**Agent:** auth-agent

**Commit message:** `feat(auth): add one-time token plugin for cross-domain handoff`

---

## Description

Add the Better Auth one-time token plugin on the server and client so the impersonation handoff can pass the session across domains. The OTT plugin generates a short-lived, single-use token tied to the current session; the operator handoff page verifies it and receives the session cookie (Better Auth PR #3659 sets cookie automatically on verify).

**Server:** Add `oneTimeToken()` from `better-auth/plugins/one-time-token` to the auth config.  
**Client:** Add `oneTimeTokenClient()` from `better-auth/client/plugins` to the shared auth client.

Use default `expiresIn` (3 minutes). `trustedOrigins` already includes `CORS_ORIGIN_ADMIN` and `CORS_ORIGIN_OPERATOR` — no changes needed there.

## Context from Discovery

### Requirements (00-requirements.md)

- **Generate:** `authClient.oneTimeToken.generate()` — requires active session; returns `{ token: string }`; default TTL 3 min.
- **Verify:** `authClient.oneTimeToken.verify({ token })` — consumes token; sets session cookie (PR #3659); returns session data.
- Single shared auth backend at `BETTER_AUTH_URL`. Both admin and operator frontends call the same auth routes.

### Plan (03-plan.md)

- Shared auth client (`packages/auth/src/client.ts`) is used by both admin and operator; adding `oneTimeTokenClient()` there covers both.
- OTT is single-use and short-lived (3 min default); handoff link is safe to use once.

## Implementation Reference

### Better Auth (Context7 + Web)

**Server plugin:**
```typescript
import { oneTimeToken } from "better-auth/plugins/one-time-token";

export const auth = betterAuth({
  // ...existing config
  plugins: [
    admin(),
    oneTimeToken(),  // add after admin()
    organization({ ... }),
  ],
});
```

**Client plugin:**
```typescript
import { oneTimeTokenClient } from "better-auth/client/plugins";

export const authClient = createAuthClient({
  baseURL: env.VITE_SERVER_URL,
  plugins: [
    adminClient(),
    oneTimeTokenClient(),
  ],
});
```

**Client API:**
- `authClient.oneTimeToken.generate()` → `{ token: string }`
- `authClient.oneTimeToken.verify({ token })` → session data; sets cookie on response
