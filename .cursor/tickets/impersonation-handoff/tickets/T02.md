## Description

Implement the Users page and impersonation handoff flow. Create `/users` under `_admin` layout with a table of users and an "Open as user" button per row. On click, run: (a) `impersonateUser`, (b) `oneTimeToken.generate`, (c) open new tab to operator handoff URL, (d) `stopImpersonating`. Handle loading, success, and error states per 01-ui-spec.

## Context from Discovery

### UI Spec (01-ui-spec.md)

**Button states:**
- **Idle:** Button enabled, "Open as user" or "View operator dashboard"
- **Loading:** Button disabled, spinner or "Opening..."
- **Success:** New tab opened; button returns to idle; admin stays on page
- **Error:** Toast or inline error; button re-enabled; "Could not open operator dashboard. Please try again."

**Components:** `Button` (ghost or outline), `Loader2` spinner, Toast (Sonner) for errors.

**Placement:** Users page (`/users`) — table lists all users; each row has "Open as user" button or row actions dropdown.

**Accessibility:** `aria-busy="true"` during load; `role="alert"` on error.

### Requirements (00-requirements.md)

- Flow: `impersonateUser({ userId })` → `oneTimeToken.generate()` → `window.open(\`${VITE_OPERATOR_URL}/auth/handoff?token=${token}\`)` → `stopImpersonating()`.
- `VITE_OPERATOR_URL` should match `CORS_ORIGIN_OPERATOR` (e.g. `http://localhost:3003` in dev).

### Plan (03-plan.md)

1. **Env:** Add `VITE_OPERATOR_URL` (optional) to `packages/env/src/web.ts`; document in admin-frontend `.env.example` that it should match `CORS_ORIGIN_OPERATOR`.
2. **Route:** Create `/users` under `_admin` layout.
3. **Page:** Table from `authClient.admin.listUsers({ query: { limit, offset, sortBy, sortDirection, ... } })`. Use TanStack Query to wrap the call. Each row: user info + "Open as user" button.
4. **Flow:** On click run the 4-step flow above.
5. **Entry:** Add Users link from admin dashboard or nav.


### Flow Logic

```typescript
const handleOpenAsUser = async (userId: string) => {
  setLoadingUserId(userId);
  setError(null);
  try {
    await authClient.admin.impersonateUser({ userId });
    const { data } = await authClient.oneTimeToken.generate();
    if (!data?.token) throw new Error("No token received");
    const operatorUrl = env.VITE_OPERATOR_URL ?? window.location.origin.replace("admin", "operator");
    window.open(`${operatorUrl}/auth/handoff?token=${data.token}`, "_blank");
    await authClient.admin.stopImpersonating();
  } catch (err) {
    toast.error("Could not open operator dashboard. Please try again.");
  } finally {
    setLoadingUserId(null);
  }
};
```

### Route Structure
