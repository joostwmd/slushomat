## Description

New router `operator-purchase.ts` wired into the existing `operatorRouter`.

### Router: `operator.purchase`

#### `operator.purchase.list`

```
Input: {
  orgSlug: string,
  machineId?: string,
  businessEntityId?: string,
  startDate?: Date,
  endDate?: Date,
  limit?: number (default 100, max 500),
  cursor?: string (ISO date for cursor pagination)
}
Output: {
  id, machineId, businessEntityId, operatorProductId,
  productName, slot, amountInCents, purchasedAt, businessEntityName?
}[]
```

**Auth logic:**
1. `resolveOrgIdFromSlug(orgSlug)` → `organizationId` (use existing helper from `operator-machine-lifecycle` work, e.g. `apps/server/src/lib/org-scope.ts`).
2. `assertUserMemberOfOrg(ctx.user.id, organizationId)` → throws `FORBIDDEN` if not a member.
3. Filter `purchase.organizationId = organizationId`.
4. Apply optional `machineId`, `businessEntityId`, `startDate`, `endDate` filters.

```sql
SELECT p.*, op.name AS product_name, be.name AS business_entity_name
FROM purchase p
JOIN operator_product op ON op.id = p.operator_product_id
LEFT JOIN business_entity be ON be.id = p.business_entity_id
WHERE p.organization_id = :organizationId
  [AND p.machine_id = :machineId]
  [AND p.business_entity_id = :businessEntityId]
  [AND p.purchased_at >= :startDate]
  [AND p.purchased_at <= :endDate]
ORDER BY p.purchased_at DESC
LIMIT :limit
```
