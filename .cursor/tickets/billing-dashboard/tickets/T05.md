# T05 — Admin UI: customers list, customer detail, machine detail

**Task ID:** T05  
**Agent:** frontend-agent  
**Depends on:** T02, T04  
**Parallel with:** T06  
**Commit message:** `feat(admin): customers list, customer detail, and machine detail views`

---

## Description

Three routes (one updated, two new) plus a query-param enhancement to the contracts route.

---

### 1. `/_admin/customers` (update existing stub)

**File:** `apps/admin-frontend/src/routes/_admin/customers.tsx`

Wire `api.admin.customer.list.useQuery()` and render:

```
Customers                          [+ New Customer]

┌─────────────────────────────────────────────────┐
│ Name         │ Slug          │ Machines │ Created│
├──────────────┼───────────────┼──────────┼────────┤
│ Betrieb GmbH │ betrieb-gmbh  │    3     │ 2d ago │
│ ...          │               │          │        │
└─────────────────────────────────────────────────┘
```

- "New Customer" button → `Link to="/_admin/create-customer"`.
- Row click → `navigate({ to: "/_admin/customers/$customerId", params: { customerId: row.id } })`.
- Empty state: "No customers yet."
- Loading: table skeleton.
- Optional client-side name search input.

---

### 2. `/_admin/customers/$customerId` (new route)

**File:** `apps/admin-frontend/src/routes/_admin/customers.$customerId.tsx`

```ts
export const Route = createFileRoute("/_admin/customers/$customerId")({ ... });
```

**Data:** `api.admin.customer.get.useQuery({ organizationId: customerId })`

**Layout:**

```
← Customers   [Org Name]   [slug badge]
```

**Tabs** (use Shadcn `Tabs`):

| Tab | Content |
|-----|---------|
| **Overview** | Business entities list via `api.admin.businessEntity.listByOrganization.useQuery({ organizationId: customerId })` + "Add Entity" button (navigate to businesses page with org pre-selected, or inline sheet) |
| **Machines** | `api.admin.customer.listMachines.useQuery({ organizationId: customerId })` → table: Machine ID, Version, Contract Status badge, Deployed (yes/no). Row click → machine detail. |
| **Products** | `api.admin.operatorProduct` or equivalent list for this org — grid of product cards (read-only). Use existing admin product list pattern if available, or `admin.operatorProduct.list` if that procedure exists; otherwise note as TODO. |
| **Purchases** | `<PurchasesTable>` wired to `api.admin.purchase.list.useQuery({ organizationId: customerId })`. `showMachineColumn: true`, `showEntityColumn: true`. Filters for date range + entity. |

---

### 3. `/_admin/customers/$customerId/machines/$machineId` (new route)

**File:** `apps/admin-frontend/src/routes/_admin/customers.$customerId.machines.$machineId.tsx`

```ts
export const Route = createFileRoute(
  "/_admin/customers/$customerId/machines/$machineId"
)({ ... });
```

**Layout:**

```
← [Org Name]   Machine [machineId]   [version badge]
                                     [Configure Contract →]
```

"Configure Contract" button (outlined, right-aligned in header):
```ts
<Link
  to="/_admin/contracts"
  search={{ organizationId: customerId, machineId }}
>
  Configure Contract
</Link>
```

**Section 1 — Purchases**

`<PurchasesTable>` with:
```ts
api.admin.purchase.list.useQuery({ machineId })
showMachineColumn={false}
showEntityColumn={true}
```

**Section 2 — Slot configuration (read-only)**

Call `api.operator.machineSlot.getConfigForMachine.useQuery(...)` — **note:** this is an `operatorProcedure`. Admin frontend may not have access; if so, create a thin `admin.machineSlot.getConfig` read-only procedure (or read `machine_slot_config` via `admin.customer.getMachineSlots` — document this decision in implementation, fall back to a TODO if out of time).

Display three cards: Left / Middle / Right, showing assigned product name or "Empty".

**Section 3 — Contract**

```ts
api.admin.operatorContract.list.useQuery({ machineId, limit: 1 })
// Show only the first (most recent) result as a summary card
```

Card shows: status badge, effective date, monthly rent (EUR), revenue share (%), business entity name. If no contract: "No contract on file."

Link: "View all contracts →" → `/_admin/contracts?machineId=<machineId>`.

### 4. `/_admin/contracts` — add query param support

**File:** `apps/admin-frontend/src/routes/_admin/contracts.tsx`

Add `validateSearch`:

```ts
import { z } from "zod";

const contractsSearch = z.object({
  organizationId: z.string().optional(),
  machineId: z.string().optional(),
});

export const Route = createFileRoute("/_admin/contracts")({
  validateSearch: contractsSearch,
  component: ContractsPage,
});
```

In the component, read `Route.useSearch()` and pass `organizationId` + `machineId` as default values to the contract create form (if the form is present on this page). If the create contract form is a sheet/dialog that opens, pass them as initial values when the dialog opens automatically when params are present (`useEffect` to open dialog if params detected).
