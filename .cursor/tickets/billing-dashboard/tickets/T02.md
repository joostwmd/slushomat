## Description

Two new admin routers: `admin-customer.ts` and `admin-purchase.ts`. Both use `adminProcedure`.

### Router: `admin.customer`

#### `admin.customer.list`

```
Input:  { search?: string }   (optional name filter; client-side filter acceptable for v1)
Output: { id, name, slug, createdAt, machineCount }[]

Query logic:
  SELECT org.id, org.name, org.slug, org.created_at,
         COUNT(DISTINCT oc.machine_id) FILTER (WHERE ocv.status = 'active') AS machine_count
  FROM organization org
  LEFT JOIN operator_contract oc ON oc.organization_id = org.id
  LEFT JOIN operator_contract_version ocv ON ocv.id = oc.current_version_id
  GROUP BY org.id
  ORDER BY org.created_at DESC
```

Note: `operatorContractVersion` is typed as `any` from the model-factory — use Drizzle `sql` template for the version join if needed.

#### `admin.customer.get`

```
Input:  { organizationId: string }
Output: { id, name, slug, createdAt }
```

Returns `NOT_FOUND` if org doesn't exist.

#### `admin.customer.listMachines`

```
Input:  { organizationId: string }
Output: { machineId, versionNumber, contractStatus: "draft"|"active"|"terminated"|"none", hasOpenDeployment: boolean }[]

Query logic:
  - SELECT DISTINCT ON (machine.id) from operator_contract base
    JOIN machine + machineVersion (for versionNumber)
    JOIN operatorContractVersion (for status)
    WHERE oc.organization_id = :organizationId
    ORDER BY machine.id, ocv.created_at DESC   (latest contract per machine)
  - For each machineId, check machine_deployment WHERE ended_at IS NULL → hasOpenDeployment
```

### Router: `admin.purchase`

#### `admin.purchase.list`

```
Input: {
  organizationId?: string,
  machineId?: string,
  businessEntityId?: string,
  startDate?: Date,
  endDate?: Date,
  limit?: number (default 100, max 500),
  cursor?: string (purchasedAt ISO for cursor pagination, or offset integer)
}
Output: {
  id, machineId, organizationId, businessEntityId,
  operatorProductId, productName, slot, amountInCents, purchasedAt,
  businessEntityName?
}[]

Query logic:
  SELECT p.*, op.name AS product_name, be.name AS business_entity_name
  FROM purchase p
  JOIN operator_product op ON op.id = p.operator_product_id
  LEFT JOIN business_entity be ON be.id = p.business_entity_id
  WHERE [optional filters]
  ORDER BY p.purchased_at DESC
  LIMIT :limit
```
