# T04 — Admin UI: platform dashboard + customer + machine

**Task ID:** T04  
**Agent:** frontend-agent  
**Depends on:** T01, T02  
**Parallel with:** —  
**Commit message:** `feat(admin): analytics on dashboard and customer views`

---

## Description

| Deliverable | Detail |
|-------------|--------|
| **Platform dashboard** | Replace placeholder in `apps/admin-frontend/src/routes/_admin/dashboard.tsx` with **`AdminPlatformAnalyticsDashboard` only** — platform-wide 4-chart suite ([FR-004](../00-requirements.md)). **No** org-scoped charts on this route. |
| **Customer detail** | On customer organization page (`_admin/customers/$customerId/index.tsx` or equivalent), embed the **same `AnalyticsDashboard` as operator**, scoped to that customer’s **org** — **no** platform-wide charts here. |
| **Machine detail** | On `/_admin/customers/$customerId/machines/$machineId`, embed **`MachineAnalyticsDashboard`** for that machine — **no** entity filter; **no** platform charts. |
| **Reuse** | Import composites from `@slushomat/ui`; **no** duplicated chart implementations in apps. |
| **Data** | `admin.analytics.*` tRPC; enforce correct procedure per page (platform vs scoped). |
| **UX** | Same global time controls and tooltip rules as operator ([01-ui-spec.md](../01-ui-spec.md)). |

### Critical product rules (echo)

- Platform admin charts **only** on **`/dashboard`**.
- Customer and machine admin pages: **same** chart suites as operator, **scoped only**.
- Charts **do not** filter purchase tables.
