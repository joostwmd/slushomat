---

## Description

| Deliverable | Detail |
|-------------|--------|
| **Routers** | Add **`operator.analytics`** and **`admin.analytics`** (nested routers merged from `operatorRouter` / `adminRouter`, same pattern as `purchase`). |
| **Input shape** | `mode`: `day` \| `week` \| `month`; **anchor** period (calendar-aligned; client sends unambiguous range or anchor date per existing conventions); **filters**: `orgSlug` (operator resolves to org), optional `machineId`, optional `businessEntityId`. |
| **Machine-scoped API** | For procedures scoped to a single machine: **reject** `businessEntityId` with **400** if passed (no entity filter at machine level). |
| **Data merge** | **MV ∪ live “today”** for Berlin: closed days from materialized view; purchases for `[start_of_today_Berlin, now)` merged without **double-counting** on the date boundary. |
| **Admin platform** | Procedures under **`adminProcedure`**: platform-wide aggregates **without** org slug; no cross-customer leakage on operator routes. |
| **Output** | **Chart-ready DTOs** (time series for bar/line/area, segments for pies) — keep transforms minimal on the client. |
| **Security** | Reuse existing operator membership / machine ownership checks; operators **never** see other orgs’ data. Admins see only the org/machine implied by the route/input. |
| **Failure / degrade** | If MV stale or refresh failed, align with [00-requirements.md](../00-requirements.md) Failure State 1 (fallback + warning/logging) as minimally as needed. |

### Product rules (echo)

- **Calendar** periods in **Europe/Berlin**, not rolling windows.
- All financial aggregation stays **server-side** (SEC / requirements).

## Acceptance Criteria

| AC ID | Criterion |
|-------|-----------|
| SEC-001 | Operators only query their organization; machine routes enforce machine access. |
| SEC-002 | Admin customer-scoped procedures cannot return other customers’ data. |
| FR-002 / FR-003 / FR-004 | DTOs sufficient for org (6 charts), machine (3 charts), and admin platform (4 charts) suites. |
| FR-010 | Historical path uses MV; “today” uses live query; merge is correct. |
| FR-006 | Inputs support day/week/month + anchored calendar periods. |
