---

## Description

| Deliverable | Detail |
|-------------|--------|
| **Chart base** | Add shadcn **Chart** + **Recharts** primitives to `packages/ui` (e.g. `ChartContainer`, config, tooltip theming) per [ui.shadcn.com chart docs](https://ui.shadcn.com/docs/components/chart). |
| **Composites** | Under `packages/ui/src/composite/`: **`AnalyticsDashboard`** (6 charts), **`MachineAnalyticsDashboard`** (3 charts), **`AdminPlatformAnalyticsDashboard`** (4 charts). Match chart types in [00-requirements.md](../00-requirements.md) FR-002–FR-004. |
| **UX** | Per [01-ui-spec.md](../01-ui-spec.md): loading **skeletons**, **error boundary per chart card**, slot or prop for **“Last updated”** badge; **rich hover tooltips**; **no** coupling to purchase table filters. |
| **Data boundary** | **No tRPC** inside composites — only **props + callbacks** (e.g. `onRetry` for a chart). Parents own fetching. |
| **Mock state** | Until T01 exists, export stable **mock data** fixtures or Storybook-style defaults so T03/T04 can swap to real queries without rewriting layout. |

### Product rules (echo)

- **Hover-first** tooltips; chart clicks **must not** filter tables (FR-007 vs UI spec: implement **tooltip + keyboard focus**, not click-to-filter).
- Responsive grid: org **2×3**, machine **2×2** with three charts; mobile single column ([01-ui-spec.md](../01-ui-spec.md) §5).
