# T03 — Operator UI: purchases pages + analytics

**Task ID:** T03  
**Agent:** frontend-agent  
**Depends on:** T01, T02  
**Parallel with:** —  
**Commit message:** `feat(operator): purchases analytics charts`

---

## Description

| Deliverable | Detail |
|-------------|--------|
| **Org purchases** | On `/_protected/$orgSlug/purchases`, render **`AnalyticsDashboard` above `PurchasesTable`**. Sync **date**, **machine**, and **business entity** filters with **analytics query inputs** only — **do not** change table filter behavior; charts **do not** filter the table. |
| **Machine purchases** | On machine purchases route (`$machineId.purchases`), render **`MachineAnalyticsDashboard`** above the table. **Remove** business entity **filter and column** if still present; **date** (and machine implicit) remain. |
| **Time controls** | Single global state: **day / week / month** toggle, **prev/next**, **calendar** popover — drives **all** charts together ([01-ui-spec.md](../01-ui-spec.md)). |
| **Data** | Fetch via **tRPC** (`operator.analytics.*`). Map DTOs to composite props. |
| **Zeros** | For current week/month, **future bucket days** show **zero** (FR-008). |
| **Tooltips** | Rich hover content; no chart-driven table filtering. |

### Product rules (echo)

- **Europe/Berlin** calendar periods.
- Machine page: **no** business entity dimension.
