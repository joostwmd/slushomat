## Description

| Area | Work |
|------|------|
| **Admin refactor** | Replace `templateProductImage` usage in `admin-template-products.ts` with `product_image` + `template_product.product_image_id`. List: `template_product` left join `product_image`. Create/update/delete/confirm: upsert or replace `product_image`, set FK, run **orphan cleanup** via T02 helper when replacing or deleting. |
| **Operator router** | New nested router under `operatorRouter`, e.g. `product`: `list`, `create`, `update`, `delete`, `copyFromTemplate`, `requestImageUpload`, `confirmProductImage`, `listTemplates`. |

| **`listTemplates`** | Read-only global templates (`organizationId` null / post-T01 global rule); still requires **`orgSlug` + membership** to call (no org filter on template rows). |
| **Image upload** | Reuse `TEMPLATE_PRODUCT_IMAGE_MAX_BYTES`, JPEG/PNG allowlist, Zod `fileSizeBytes` cap from admin; operator paths use **`operatorProductId`** and a stable path prefix (e.g. `operator-products/{orgId or productId}/…`). **Never** trust client full paths on confirm. |
| **Registration** | Mount new router on `operator.ts`. |

### Requirements (`00-requirements.md`)

- Operators cannot mutate `template_product` through operator routes.
- Field parity and validation (tax 7/19, image MIME/size) match template products.
- Shared image: multiple products may reference the same `product_image.id`.

### Plan (`03-plan.md`)

- Admin delete flow must use reference counting before removing Storage objects.
- Constants may move to `@slushomat/db` or `apps/server/src/lib/product-image.ts` if duplicated.
