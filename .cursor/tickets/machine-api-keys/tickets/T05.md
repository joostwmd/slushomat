## Product rules (carry through implementation)

1. **Keys:** Better Auth plugin, prefix **`SLUSH_`**, **`configId: "machine"`**, **`metadata.machineId`**, **`userId` = authenticated admin** on `createApiKey` / rotate.
2. **One key per machine** — rotate deletes/replaces previous; revoke clears **`machine.apiKeyId`**.
3. **Never** return full plaintext secret except create/rotate responses; list/get metadata only (prefix, start, enabled, ids).
4. **Machine server:** Bearer + **`X-Machine-Id`** → `verifyApiKey` → match **`metadata.machineId`** to header → load **`machine`** → **`disabled` → 403** `MACHINE_DISABLED`, else invalid → **401** `INVALID_MACHINE_CREDENTIALS`; **`/healthz`** unauthenticated.
5. Shared **`auth`** from **`@slushomat/auth`** in machine-server (add workspace dependency + **`DATABASE_URL`**, **`BETTER_AUTH_*`** env).

## Description

Implement **admin-only** tRPC procedures for **create / rotate / revoke** machine API keys and a **metadata** query (prefix, **`start`**, enabled state, ids, timestamps — **never** the full plaintext secret except on create/rotate responses). Enforce **one key per machine**: rotation **deletes or invalidates** the previous Better Auth key and updates **`machine.apiKeyId`**; revoke clears **`machine.apiKeyId`**. Resolve keys **by `machineId` server-side** (join **`machine.apiKeyId`** with **`getApiKey`** / DB) so **any admin** sees the correct state, not only keys whose **`userId`** matches the session. Implement **`machineAuthMiddleware`** in **`apps/machine-server`**: parse **`Authorization: Bearer`**, read **`X-Machine-Id`**, call **`auth.api.verifyApiKey`** with **`configId: "machine"`**, assert **`metadata.machineId`** matches header, load **`machine`** row, return **401** + **`INVALID_MACHINE_CREDENTIALS`** if invalid/mismatch/revoked, **403** + **`MACHINE_DISABLED`** if **`disabled`**. **Do not** require auth on **`GET /healthz`**; protect **`/is-killed`** (and future machine routes) with scoped middleware. Wire **`machine-server`** to **`@slushomat/auth`** (and **`@slushomat/db`** for machine lookup) with **env parity** (`DATABASE_URL`, `BETTER_AUTH_SECRET`, `BETTER_AUTH_URL` per `packages/env` / `apps/server`).

## Implementation steps

1. **tRPC — metadata query**  
   Input: **`machineId`**. Load **`machine`**; if **`apiKeyId`** is null, return **`null`** (no key). Else fetch key via **`auth.api.getApiKey`** (or DB select on **`apikey`** table) filtered to **`configId: "machine"`** if needed. Output: **`{ id, prefix, start, enabled, createdAt, updatedAt }`** — **omit `key`**.

2. **tRPC — create**  
   Input: **`machineId`**. If **`machine.apiKeyId`** is non-null → **409** or instruct client to use rotate. Else **`auth.api.createApiKey`** with **`userId: ctx.user.id`**, **`metadata: { machineId }`**, **`configId: "machine"`** (or per-plugin API). Update **`machine.apiKeyId`** to new id. Return **plaintext `key` once** in output (narrow Zod) + metadata.

3. **tRPC — rotate**  
   Confirm old secret invalid **immediately**: **delete** old api key (or plugin **revoke** then create). **`createApiKey`** again; set **`machine.apiKeyId`** to new id. Return new **plaintext `key`** once.

4. **tRPC — revoke**  
   If **`apiKeyId`** set → **`auth.api.deleteApiKey`** (or equivalent); **`update machine set apiKeyId = null`**. Idempotent if already null.

5. **Cross-admin listing**  
   Never rely on “list keys for session user only”; always anchor on **`machineId`** + **`machine.apiKeyId`** / metadata query.

6. **machine-server middleware**  
   - Extract Bearer secret (reuse same parsing logic mentally as **`customAPIKeyGetter`** in T03).  
   - Read **`X-Machine-Id`** (trim; reject missing).  
   - **`verifyApiKey`**; on failure → **401** with **`MACHINE_ERROR_CODES.INVALID_MACHINE_CREDENTIALS`**.  
   - Compare **`String(metadata.machineId)`** to **`String(headerMachineId)`**; mismatch → **401** (same code).  
   - **`ctx.db` / `db`**: `select * from machine where id = ?`; not found → **401**; **`disabled`** → **403** **`MACHINE_DISABLED`**.  
   - **`c.set("machineId", id)`** (or minimal context) for downstream handlers.

7. **`index.ts` routing**  
   Register **`GET /healthz`** **before** any auth middleware, **or** mount **`machineAuthMiddleware`** only on **`/is-killed`**. Do **not** log the bearer token.

8. **Types / Zod**  
   Use separate output schemas: **`MachineApiKeySecretPayload`** (includes **`key: string`**) vs **`MachineApiKeyPublicMeta`** (no **`key`**).

### 1) Parse Bearer (shared helper — optional `packages/api` or local)

```typescript
export function parseBearerAuthorization(header: string | undefined): string | null {
  if (!header) return null;
  const m = /^Bearer\s+(.+)$/i.exec(header.trim());
  const raw = m?.[1]?.trim();
  return raw || null;
}
```

### 2) `machine-auth.ts` — Hono middleware (core flow)

```typescript
import { createMiddleware } from "hono/factory";
import { HTTPException } from "hono/http-exception";
import { eq } from "drizzle-orm";
import { auth } from "@slushomat/auth";
import { db } from "@slushomat/db";
import { machine } from "@slushomat/db/schema";
import { MACHINE_ERROR_CODES } from "../errors";

function unauthorized() {
  return new HTTPException(401, {
    message: JSON.stringify({ code: MACHINE_ERROR_CODES.INVALID_MACHINE_CREDENTIALS }),
  });
}

function forbiddenDisabled() {
  return new HTTPException(403, {
    message: JSON.stringify({ code: MACHINE_ERROR_CODES.MACHINE_DISABLED }),
  });
}

export const machineAuthMiddleware = createMiddleware(async (c, next) => {
  const secret = parseBearerAuthorization(c.req.header("authorization"));
  const machineIdHeader = c.req.header("x-machine-id")?.trim();
  if (!secret || !machineIdHeader) throw unauthorized();

  const verified = await auth.api.verifyApiKey({
    body: { key: secret, configId: "machine" },
    headers: c.req.raw.headers,
  });
  if (!verified?.key) throw unauthorized(); // adjust to actual verifyApiKey return shape

  const meta = verified.key.metadata as { machineId?: string } | undefined;
  if (!meta?.machineId || String(meta.machineId) !== String(machineIdHeader)) {
    throw unauthorized();
  }

  const [row] = await db
    .select()
    .from(machine)
    .where(eq(machine.id, machineIdHeader))
    .limit(1);
  if (!row) throw unauthorized();
  if (row.disabled) throw forbiddenDisabled();

  c.set("machineId", row.id);
  await next();
});
```

*Adjust **`verified.key`** / **`verifyApiKey`** result handling to match the real Better Auth server API; use **`if (!verified || verified.error)`** patterns from docs.*

### 4) tRPC — nested `apiKey` router (repo style: `admin-machines.ts`)

import { z } from "zod";
import { eq } from "drizzle-orm";
import { auth } from "@slushomat/auth";
import { machine } from "@slushomat/db/schema";
import { router } from "../init";
import { adminProcedure } from "../procedures";

const apiKeySecretOut = z.object({
  machineId: z.string(),
  apiKeyId: z.string(),
  key: z.string(), // full secret — only create/rotate
  prefix: z.string().optional(),
  start: z.string().nullable().optional(),
});

const apiKeyMetaOut = z
  .object({
    id: z.string(),
    prefix: z.string().nullable().optional(),
    start: z.string().nullable().optional(),
    enabled: z.boolean(),
    createdAt: z.date(),
    updatedAt: z.date(),
  })
  .nullable();

export const machineApiKeyAdminRouter = router({
  getMetadata: adminProcedure
    .input(z.object({ machineId: z.string().min(1) }))
    .output(apiKeyMetaOut)
    .query(async ({ ctx, input }) => {
      const [m] = await ctx.db
        .select({ apiKeyId: machine.apiKeyId })
        .from(machine)
        .where(eq(machine.id, input.machineId))
        .limit(1);
      if (!m?.apiKeyId) return null;
      const rec = await auth.api.getApiKey({
        body: { id: m.apiKeyId },
        headers: ctx.headers,
      });
      if (!rec) return null;
      return {
        id: rec.id,
        prefix: rec.prefix ?? null,
        start: rec.start ?? null,
        enabled: rec.enabled !== false,
        createdAt: new Date(rec.createdAt),
        updatedAt: new Date(rec.updatedAt),

## Pitfalls / notes

- **`verifyApiKey` return value**: Do not assume property names; align with **T03** plugin **`configId`**.  
- **401 vs 403**: Wrong **`X-Machine-Id`** pairing is **401** (invalid credentials), not 403.  
- **Never** `console.log` headers or parsed bearer secrets.  
- **HTTPException message**: If the app expects JSON **`{ code }`**, use **`c.json(..., status)`** inside middleware instead of **`message: JSON.stringify`** if that fits existing **`onError`** handling — align with how **`HTTPException`** is handled in **`index.ts`** today.  
- **Session forwarding**: Some **`auth.api.*`** calls need **`headers: ctx.headers`** from tRPC context (see **`createContext`** in `apps/server/src/trpc/context.ts`).  
- **Machine server `AppEnv` typing**: If you **`c.set("machineId", ...)`**, extend Hono **`Variables`** type for type-safe handlers.
