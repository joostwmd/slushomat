## Product rules (reflect in UX; backend enforces)

1. **Keys:** Prefix **`SLUSH_`**, **`configId: "machine"`** (device config copy should show **machine id** + Bearer secret); issuance is admin-only (T05).
2. **One key per machine** — UI offers **Generate** (no key) vs **Rotate** (has key); **Revoke** clears linkage.
3. **Never** show full secret after modal dismiss — only prefix + **`start`** (OQ-1 **A**).
4. **Machine server** pairing is Bearer + **`X-Machine-Id`** (document in operator/device runbooks if needed); not implemented in this ticket.
5. Admin app uses the same **`auth`** session as today; no duplicate Better Auth client config for keys beyond tRPC.

## Description

Extend the **admin machines** experience in **`machines.tsx`**: inside the **machine create/edit sheet** (and/or a clearly associated section), implement **Generate API key**, **Rotate API key** (with confirmation that the **old secret stops working immediately**), and **Revoke**. On **create/rotate**, show a **one-time modal** with the **full secret** (monospace), **Copy** control, helper copy (“Copy this now.”), and the **machine id** for device configuration. After dismiss, show only a **persistent “Device API key”** row: **prefix** + masked **`start`**, **enabled** state, timestamps — **no** full secret (per **OQ-1 A** / Better Auth). Use **`useMutation`** + **`invalidateQueries`** / **`refetch`** so lists and metadata stay fresh. Surface errors with **`toast`** (existing pattern on this page). Meet basic **a11y** for the dialog (label secret region, focus management via Radix/shadcn defaults).

## Implementation steps

1. Add **`useQuery`** for **`trpc.admin.machine.apiKey.getMetadata`** with **`machineId: machineEditId`**, **`enabled: !!machineEditId && machineSheetOpen`** (avoid fetching without an id).
2. **No key** (`metadata` null): show **Generate API key** button → **`create`** mutation → on success open **secret modal** with returned **`key`**.
3. **Has key**: show **identity row** + **Rotate** / **Revoke**. Rotate → **`window.confirm`** or **`AlertDialog`** with explicit “old secret invalid immediately” → **`rotate`** mutation → show modal with new **`key`**. Revoke → confirm → **`revoke`** mutation → clear UI to “no key” state.
4. On modal **Dismiss**: clear local state holding **`plaintextKey`**; rely on **`getMetadata`** for masked display only.
5. **`onSuccess`** for each mutation: **`void machinesQuery.refetch()`** and **`queryClient.invalidateQueries`** for the metadata query (use **`trpc.admin.machine.apiKey.getMetadata.queryFilter()`** if available in your tRPC v11 helpers, else explicit **`queryKey`**).
6. Ensure **Create machine** flow: after creating a machine, user may need to **re-open** edit or stay on sheet — if **`machineEditId`** is set only after create, chain **open edit** or pass **new id** into key section (follow existing **`onSuccess`** pattern from **`createMachineMutation`**).

## Code examples

### 1) Metadata query + mutations (TanStack Query + tRPC — matches `machines.tsx`)

```tsx
import { useMutation, useQuery, useQueryClient } from "@tanstack/react-query";
import { trpc } from "@/utils/trpc";

function useMachineApiKey(machineId: string | null, sheetOpen: boolean) {
  const qc = useQueryClient();

  const metaQuery = useQuery({
    ...trpc.admin.machine.apiKey.getMetadata.queryOptions(
      { machineId: machineId! },
      { enabled: sheetOpen && !!machineId },
    ),
  });

  const invalidate = () => {
    if (!machineId) return;
    void qc.invalidateQueries(
      trpc.admin.machine.apiKey.getMetadata.queryFilter({ machineId }),
    );
  };

  const createKey = useMutation({
    ...trpc.admin.machine.apiKey.create.mutationOptions(),
    onSuccess: () => {
      invalidate();
    },
  });

  const rotateKey = useMutation({
    ...trpc.admin.machine.apiKey.rotate.mutationOptions(),
    onSuccess: () => {
      invalidate();
    },
  });

  const revokeKey = useMutation({
    ...trpc.admin.machine.apiKey.revoke.mutationOptions(),
    onSuccess: () => {
      invalidate();
    },
  });

  return { metaQuery, createKey, rotateKey, revokeKey };
}
```

*Adjust procedure names (**`rotate`/`revoke`**) to match **T05** implementation.*

### 2) One-time secret modal (shadcn Dialog sketch)

```tsx
import { useState } from "react";
import { Button } from "@slushomat/ui/base/button";
import {
  Dialog,
  DialogContent,
  DialogDescription,
  DialogFooter,
  DialogHeader,
  DialogTitle,
} from "@slushomat/ui/base/dialog";
import { toast } from "sonner";

function SecretOnceDialog(props: {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  machineId: string;
  secret: string | null;
}) {
  const { open, onOpenChange, machineId, secret } = props;

  const copySecret = async () => {
    if (!secret) return;
    await navigator.clipboard.writeText(secret);
    toast.success("API key copied");
  };

  return (
    <Dialog open={open} onOpenChange={onOpenChange}>
      <DialogContent className="sm:max-w-lg" aria-describedby="machine-api-key-desc">
        <DialogHeader>
          <DialogTitle>Save this API key</DialogTitle>
          <DialogDescription id="machine-api-key-desc">
            Copy this now. You won&apos;t be able to see the full key again. If you lose it,
            rotate the key from the machine settings.
          </DialogDescription>
        </DialogHeader>
        <div className="flex flex-col gap-2">
          <p className="text-xs text-muted-foreground">
            Machine id{" "}
            <span className="font-mono text-foreground">{machineId}</span>
          </p>
          <label className="text-xs font-medium" htmlFor="machine-api-key-secret">
            API key
          </label>
          <textarea
            id="machine-api-key-secret"
            readOnly
            className="min-h-[88px] w-full resize-none border border-input bg-muted/30 p-2 font-mono text-xs"
            value={secret ?? ""}
          />
        </div>
        <DialogFooter className="gap-2 sm:gap-0">
          <Button type="button" variant="outline" onClick={copySecret} disabled={!secret}>
            Copy
          </Button>
          <Button type="button" onClick={() => onOpenChange(false)}>
            Done
          </Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
```

### 3) Persistent identity row inside the machine **Sheet**

```tsx
function DeviceApiKeySection(props: {
  machineId: string | null;
  sheetOpen: boolean;
}) {
  const [secretModal, setSecretModal] = useState<string | null>(null);
  const { metaQuery, createKey, rotateKey, revokeKey } = useMachineApiKey(
    props.machineId,
    props.sheetOpen,
  );

  if (!props.machineId) {
    return (
      <p className="text-xs text-muted-foreground">
        Save the machine first to manage its API key.
      </p>
    );
  }

  const meta = metaQuery.data;

  return (
    <div className="mt-6 border-t border-border pt-4">
      <h3 className="mb-2 text-sm font-medium">Device API key</h3>
      {!meta ? (
        <Button
          type="button"
          size="sm"
          disabled={createKey.isPending}
          onClick={() =>
            createKey.mutate(
              { machineId: props.machineId! },
              {
                onSuccess: (data) => setSecretModal(data.key),
                onError: (e) => toast.error(errMessage(e)),
              },
            )
          }
        >
          Generate API key
        </Button>
      ) : (
        <div className="flex flex-col gap-2 text-xs">
          <p className="font-mono text-[11px]">
            {(meta.prefix ?? "SLUSH_") + (meta.start ? `${meta.start}***` : "****")}
          </p>
          <p className="text-muted-foreground">
            {meta.enabled ? "Enabled" : "Disabled"} · updated{" "}
            {meta.updatedAt.toLocaleString()}
          </p>
          <div className="flex flex-wrap gap-2">
            <Button
              type="button"
              size="sm"
              variant="secondary"
              disabled={rotateKey.isPending}
              onClick={() => {
                if (
                  !confirm(
                    "Rotate this key? The old secret stops working immediately.",
                  )
                )
                  return;
                rotateKey.mutate(
                  { machineId: props.machineId! },
                  {
                    onSuccess: (data) => setSecretModal(data.key),
                    onError: (e) => toast.error(errMessage(e)),
                  },
                );
              }}
            >
              Rotate key
            </Button>
            <Button
              type="button"
              size="sm"
              variant="destructive"
              disabled={revokeKey.isPending}
              onClick={() => {
                if (!confirm("Revoke this key? Devices using it will stop authenticating."))
                  return;
                revokeKey.mutate(
                  { machineId: props.machineId! },
                  { onError: (e) => toast.error(errMessage(e)) },
                );
              }}
            >
              Revoke
            </Button>
          </div>
        </div>
      )}

      <SecretOnceDialog
        open={!!secretModal}
        onOpenChange={(o) => {
          if (!o) setSecretModal(null);
        }}
        machineId={props.machineId}
        secret={secretModal}
      />
    </div>
  );
}
```

*Wire **`errMessage`** from the existing helper in **`machines.tsx`**. For **new machines**, ensure **`machineEditId`** is populated after **`createMachineMutation`** **`onSuccess`** if you want keys during first save — adjust the create-machine flow accordingly.*

## Pitfalls / notes

- **Never** persist the plaintext **`key`** in React state beyond the modal’s lifetime; clearing on dismiss avoids accidental leakage via devtools screenshots.  
- **Procedure availability**: If **T05** names differ (**`regenerate`** vs **`rotate`**), rename UI only — keep one product term in copy (**Rotate**).  
- **Loading / error states**: Show **`metaQuery.isPending`** skeleton or “Loading key info…”.  
- **Disabled machines**: If **T05** exposes **`disabled`** on **`machine.list`**, optionally show a banner that keys still exist but device is blocked — optional polish, not required for minimal pass.  
- **Full key “reveal” later**: Explicitly **do not** implement — not supported without rotate.
