## Description

Ship the **Products** admin experience:

1. **Route:** TanStack Router file route [`apps/admin-frontend/src/routes/_admin/products.tsx`](apps/admin-frontend/src/routes/_admin/products.tsx) → URL such as `/_admin/products` (match project’s path style; requirements suggest `/products` under admin layout).
2. **Navigation:** Entry in [`apps/admin-frontend/src/components/admin-app-sidebar.tsx`](apps/admin-frontend/src/components/admin-app-sidebar.tsx) and crumbs in [`apps/admin-frontend/src/components/admin-breadcrumbs.tsx`](apps/admin-frontend/src/components/admin-breadcrumbs.tsx).
3. **UI:** List + **Sheet**-based create/edit patterned on [`apps/admin-frontend/src/routes/_admin/machines.tsx`](apps/admin-frontend/src/routes/_admin/machines.tsx) (`useQuery` / `useMutation`, `trpc`, `toast`, `@slushomat/ui` primitives).
4. **Image:** Install **Dice UI** file upload via shadcn registry: `npx shadcn@latest add @diceui/file-upload` — compose with existing tokens.
5. **Upload flow:** Persist template product (if needed) → **`requestImageUpload`** on `trpc.admin.templateProduct` → create **anon** Supabase client with `VITE_SUPABASE_*` → `storage.from(bucket).uploadToSignedUrl(...)` → **`confirmTemplateProductImage`** (or equivalent from T03) → invalidate queries.
6. **Display:** Thumbnail or placeholder in list; full preview in sheet. If bucket is **private**, use **signed read** from API (add procedure in T03 if not already) or public URL if bucket is public — align with human checkpoint 1.

Admin layout guard remains in [`apps/admin-frontend/src/routes/_admin/route.tsx`](apps/admin-frontend/src/routes/_admin/route.tsx) (no change required unless adding redirect from dashboard).

### Supabase client in Vite

- Use env from [`packages/env/src/web.ts`](packages/env/src/web.ts) after T01 adds `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`.
- **Never** import server env or service role in admin-frontend.

```ts
import { createClient } from "@supabase/supabase-js";

const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_ANON_KEY,
);
// After tRPC returns { bucket, path, token }:
await supabase.storage.from(bucket).uploadToSignedUrl(path, token, file, {
  contentType: file.type,
});
```

Official: [createSignedUploadUrl](https://supabase.com/docs/reference/javascript/storage-from-createsigneduploadurl) (server mint + client upload).
