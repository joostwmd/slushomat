## Description

Add Drizzle schema and a generated migration for **template products** (global catalog line items):

1. **`template_product`** — template line item: `id` (text PK), `name`, `price`, `tax_rate_percent` restricted to **7 or 19**, optional nullable **`organization_id`** for future operator scoping, `created_at`, `updated_at`.
2. **`template_product_image`** — links a template product to Storage: `id` (text PK), `template_product_id` FK → `template_product.id` **`onDelete: cascade`**, `bucket` (text, not null), `object_path` (text, not null).

**MVP single image:** enforce **at most one row per `template_product_id`** via `unique (template_product_id)` *or* equivalent constraint so the API/UI can treat “primary image” unambiguously. (Plan allows `sort_order` later for multi-image.)

Use **text PKs** and `crypto.randomUUID()` (or project standard) at insert time, consistent with [`packages/db/src/schema/machines.ts`](packages/db/src/schema/machines.ts).

In TypeScript, export Drizzle table objects as e.g. **`templateProduct`** and **`templateProductImage`** (camelCase) mapping to the SQL names above.

Export new tables from [`packages/db/src/schema/index.ts`](packages/db/src/schema/index.ts). Run **`db:generate`** / migrate per [`packages/db/drizzle.config.ts`](packages/db/drizzle.config.ts).

  id: text("id").primaryKey(),
