## Description

Introduce a **`packages/supabase`** workspace package (`@slushomat/supabase`) that:

1. Exposes a **server** Supabase client factory using the **service role** key, **`persistSession: false`** (no browser session storage on server).
2. Exports a **`SupabaseStorageService`** (or equivalent module) with:
   - **`createSignedUploadUrl`** (wrap `storage.from(bucket).createSignedUploadUrl(path, options?)`).
   - **`removeObjects`** (delete storage objects on template product delete — per agreed policy from human checkpoint 1).
   - Optional **`createSignedUrl`** (or similar) for **private bucket** thumbnail reads if the team chooses private objects.

Wire **`apps/server`** to depend on `@slushomat/supabase` and add **`@supabase/supabase-js`** where needed. Extend **typed env** so server and Vite client get the right variables **without** leaking the service role into the admin bundle.

### Supabase JS — signed upload (official)

- **Docs:** [Storage: `createSignedUploadUrl`](https://supabase.com/docs/reference/javascript/storage-from-createsigneduploadurl)
- Server (service role): `storage.from(bucket).createSignedUploadUrl(path, options?)` returns path + token for a time-limited upload.
- Browser (anon client): `uploadToSignedUrl(path, token, file, { contentType })` — token from the server response.

**Illustrative shape (reference only):**

```ts
// Server: mint URL (service role client)
const { data, error } = await supabase.storage
  .from(bucket)
  .createSignedUploadUrl(objectPath);

// Client: upload file with returned token (anon client — never use service role here)
await supabase.storage
  .from(bucket)
  .uploadToSignedUrl(objectPath, token, file, { contentType: file.type });
```
