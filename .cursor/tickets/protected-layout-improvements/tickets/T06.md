## Description

Restructure the operator app to support org-scoped routing, pending invitations, org selection, and an updated handoff flow. This is the most complex ticket and touches multiple flows.

**Step-by-step implementation:**

1. **_protected beforeLoad Б─■ org gate:** After session check, call `authClient.organization.list()`. If empty (user has no orgs), redirect to `/invitations` unless already on `/invitations`. If session exists and org list is non-empty, allow access. Handle `/invitations` and `/organizations` as special routes that are always allowed within _protected (no org required for invitations; org selection is the destination when slug mismatch).

2. **Create /invitations route:** New route `_protected/invitations.tsx`. Use `authClient.organization.listUserInvitations()` to fetch pending invitations. Display list with accept/reject actions. `authClient.organization.acceptInvitation({ invitationId })` and `rejectInvitation` (or equivalent). After accept, user gains org membership; redirect to `/organizations` or `/${slug}/dashboard` after refresh.

3. **Create /organizations route:** New route `_protected/organizations.tsx`. Lists user's orgs via `organization.list()`. Each org has `id`, `slug`, `name`. On select: call `authClient.organization.setActive({ organizationId, organizationSlug })`, then redirect to `/${organizationSlug}/dashboard`. Use Link or button with `navigate({ to: `/${slug}/dashboard` })` after setActive.

4. **Create _protected.$orgSlug layout:** Dynamic layout route with path segment `$orgSlug`. Route file: `_protected.$orgSlug/route.tsx`. In `beforeLoad`: compare `params.orgSlug` with `session.data?.session?.activeOrganizationSlug`. If mismatch (or missing), redirect to `/organizations`. Otherwise allow access.
   - Child routes: `_protected.$orgSlug/index.tsx` (redirect to `/${orgSlug}/dashboard`), `_protected.$orgSlug/dashboard.tsx`.
   - Move current `_protected/dashboard` and `_protected/index` content under `_protected.$orgSlug` or create new files. Delete or update old routes.
   - Route structure: `_protected` Б├▓ `invitations`, `organizations`, `_protected.$orgSlug` (layout) Б├▓ `index`, `dashboard`.

5. **OperatorAppSidebar:** Create `apps/operator-frontend/src/components/operator-app-sidebar.tsx`. Compose from `packages/ui` base: Sidebar, SidebarContent, SidebarFooter, NavMain (minimal placeholder links), NavUser. Use NavUser with `impersonated={!!session?.data?.session?.impersonatedBy}`, `onStopImpersonating={() => authClient.admin.stopImpersonating()}`, `onSignOut={() => authClient.signOut()}`. No TeamSwitcher.

6. **Layout structure:** The `_protected.$orgSlug` layout (or _protected) wraps with SidebarProvider + OperatorAppSidebar + SidebarInset + Outlet. Decide where the shell lives: (a) _protected layout shows shell for all children including invitations, organizations, and $orgSlug, or (b) only $orgSlug layout shows the sidebar. Per plan: "Layout: SidebarProvider + OperatorAppSidebar + SidebarInset + Outlet" for operator. Likely the sidebar is shown for invitations, organizations, and org-scoped pages. So _protected layout provides the shell.

7. **Handoff update:** In `apps/operator-frontend/src/routes/auth/handoff.tsx`, after `authClient.oneTimeToken.verify({ token })` succeeds: call `authClient.organization.list()`, take `data[0]?.slug`. If slug exists, redirect to `/${slug}/dashboard`. If no org (empty list), redirect to `/invitations`. Remove the current redirect to `/dashboard`.

8. **Slug mismatch flow:** When user visits `/:orgSlug/dashboard` and `params.orgSlug !== session.activeOrganizationSlug`, the `_protected.$orgSlug` beforeLoad redirects to `/organizations`. User selects org on that page Б├▓ setActive Б├▓ redirect to `/${slug}/dashboard`.

9. **Route file structure (TanStack Router file-based):** TanStack Router uses file names for routes. `_protected.$orgSlug` suggests a layout with dynamic segment. Check router conventions: `_protected.$orgSlug/route.tsx` for the layout, `_protected.$orgSlug/dashboard.tsx` for the dashboard. The path would be `/:orgSlug/dashboard`. Ensure the route tree generates correctly.

### TanStack Router Dynamic Params, beforeLoad

```tsx
// _protected.$orgSlug/route.tsx
export const Route = createFileRoute("/_protected/$orgSlug")({
  beforeLoad: async ({ params }) => {
    const session = await authClient.getSession();
    if (!session.data) throw redirect({ to: "/sign-in" });
    if (params.orgSlug !== session.data.session?.activeOrganizationSlug) {
      throw redirect({ to: "/organizations" });
    }
    return { orgSlug: params.orgSlug };
  },
  component: OrgSlugLayout,
});
```

**File naming:** TanStack Router file-based routing uses `$param` for dynamic segments. A folder `_protected.$orgSlug` creates a layout with param `orgSlug`. Child routes inherit the param. Verify exact route ID: may be `/_protected/$orgSlug` or `/_protected/$orgSlug/`.

### Handoff flow

```typescript
authClient.oneTimeToken.verify({ token }).then(async () => {
  const { data } = await authClient.organization.list();
  const slug = data?.[0]?.slug;
  if (slug) {
    navigate({ to: `/${slug}/dashboard` });
  } else {
    navigate({ to: "/invitations" });
  }
});
```

## Risk / Notes

- **Route structure:** TanStack Router file-based routing varies by version. `_protected.$orgSlug` might create path `/_protected/:orgSlug` or `/:orgSlug` depending on pathless layouts. The plan expects URLs `/:orgSlug/dashboard`. Ensure _protected is pathless (`_` prefix) and $orgSlug adds the slug to the path. May need to adjust route IDs.
- **Session refresh:** After `setActive`, the session updates. The redirect to `/${slug}/dashboard` should use the new activeOrganizationSlug. If there's a brief delay, the $orgSlug layout's beforeLoad might run before session sync Б─■ consider if a short loading state or refetch is needed.
- **Invitations flow:** After acceptInvitation, the user is a member. Redirect to org selection or directly to `/${newOrgSlug}/dashboard`. The new org might need to be set active Б─■ check if acceptInvitation auto-sets it.
- **Auth client:** Operator app must use authClient with `organizationClient()` (T03). Ensure operator-frontend imports from `@slushomat/auth/client`.
- **Index redirect:** `_protected/index` might redirect to dashboard. With org routing, the index could redirect to `/organizations` (if no active slug) or `/${activeSlug}/dashboard`. The plan says: `_protected` index Б├▓ redirect /dashboard; but with org routing, there's no single /dashboard. Update: `_protected/index` Б├▓ redirect to `/organizations` (user picks org) or to `/${activeSlug}/dashboard` if active org exists. Simplify: index redirects to `/organizations` always, or check session.activeOrganizationSlug and redirect to `/${slug}/dashboard` if set.
