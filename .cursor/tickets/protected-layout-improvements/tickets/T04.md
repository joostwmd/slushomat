## Description

Extend the `NavUser` component in `packages/ui/src/base/nav-user.tsx` so that both admin and operator apps can render context-dependent user navigation without importing the auth client. The component must accept `onSignOut` (required), and optionally `impersonated` and `onStopImpersonating`. When `impersonated=true`, show "Stop impersonating" and call `onStopImpersonating` on click; otherwise show "Sign out" and call `onSignOut`. Remove the hardcoded "Log out" and sample dropdown items (Upgrade to Pro, Account, Billing, Notifications) that are not part of the requirements — replace with the single contextual action.

**Step-by-step implementation:**

1. Add props to NavUser: `user` (existing), `onSignOut: () => void | Promise<void>` (required), `impersonated?: boolean` (default false), `onStopImpersonating?: () => void | Promise<void>` (optional).
2. In the dropdown, when `impersonated` is true and `onStopImpersonating` is provided, render a single `DropdownMenuItem` with "Stop impersonating" (and an appropriate icon, e.g. `UserX` or similar) that calls `onStopImpersonating` on click.
3. Otherwise, render a single `DropdownMenuItem` with "Sign out" (and `LogOutIcon`) that calls `onSignOut` on click.
4. Remove the sample items: Upgrade to Pro, Account, Billing, Notifications, and the generic "Log out". The footer section should only show the contextual action (Sign out or Stop impersonating).
5. Ensure the component does NOT import `@slushomat/auth/client` or any auth package — all behavior is driven by props.
6. Keep the user avatar/name/email display and the `DropdownMenu` structure. Simplify the dropdown content to: user info block (optional, can keep for consistency) + separator + the single action item.

## Research / Context7

### Shadcn DropdownMenu / NavUser Patterns

From Context7 (Shadcn UI):

- **DropdownMenuItem:** Use `onSelect` for click handlers. Can be async.
- **Sidebar composition:** NavUser lives in SidebarFooter; uses `SidebarMenu`, `SidebarMenuItem`, `DropdownMenu`, `DropdownMenuTrigger`, `DropdownMenuContent`, `DropdownMenuItem`.
- **Icons:** Lucide icons (`LogOutIcon` already used). Consider `UserXIcon` or `ShieldOffIcon` for "Stop impersonating".

### Avoiding Auth Client in Shared UI

- Pass callbacks from the route/layout that has access to auth. Admin: `onSignOut={() => authClient.signOut()}`. Operator: `impersonated={!!session?.data?.session?.impersonatedBy}`, `onStopImpersonating={() => authClient.admin.stopImpersonating()}`, `onSignOut={() => authClient.signOut()}`.

## Implementation Reference

```tsx
// packages/ui/src/base/nav-user.tsx
import { UserXIcon, LogOutIcon } from "lucide-react";

export function NavUser({
  user,
  onSignOut,
  impersonated = false,
  onStopImpersonating,
}: {
  user: { name: string; email: string; avatar?: string };
  onSignOut: () => void | Promise<void>;
  impersonated?: boolean;
  onStopImpersonating?: () => void | Promise<void>;
}) {
  const { isMobile } = useSidebar();
  const handleAction = impersonated && onStopImpersonating
    ? onStopImpersonating
    : onSignOut;
  const actionLabel = impersonated ? "Stop impersonating" : "Sign out";
  const ActionIcon = impersonated ? UserXIcon : LogOutIcon;

  return (
    <SidebarMenu>
      <SidebarMenuItem>
        <DropdownMenu>
          {/* ... trigger (avatar, name, email) unchanged ... */}
          <DropdownMenuContent ...>
            <DropdownMenuGroup>
              <DropdownMenuLabel>...</DropdownMenuLabel>
            </DropdownMenuGroup>
            <DropdownMenuSeparator />
            <DropdownMenuItem onSelect={() => handleAction()}>
              <ActionIcon />
              {actionLabel}
            </DropdownMenuItem>
          </DropdownMenuContent>
        </DropdownMenu>
      </SidebarMenuItem>
    </SidebarMenu>
  );
}
```
