## Description

Create a `getInitialOrganization(userId)` function in the db package that returns `{ id, slug }` for the user's first organization. The "first" org is defined as the user's earliest membership by `member.createdAt`, joined with the `organization` table to retrieve the slug. This query runs on every session creation (hot path), so it must use Drizzle's `.prepare()` for preloading — prepared statements improve performance through query plan caching and reduce overhead on repeated executions.

**Step-by-step implementation:**

1. Create `packages/db/src/queries/get-initial-organization.ts`.
2. Define the query: select from `member` joined with `organization` where `member.userId = ?`, ordered by `member.createdAt ASC`, limit 1. Return only `organization.id` and `organization.slug`.
3. Use `sql.placeholder('userId')` for the parameter and call `.prepare('get_initial_organization')` on the query.
4. Export a function `getInitialOrganization(userId: string)` that executes the prepared statement and returns `{ id, slug } | null`.
5. Export the function from `packages/db/src/index.ts` (or the appropriate public entry point).
6. Ensure indexes exist: `member_userId_idx` and `member_organizationId_idx` are already defined in `packages/db/src/schema/auth.ts`.

## Research / Context7

### Drizzle Prepared Statements (Context7: /drizzle-team/drizzle-orm)

```typescript
import { eq, sql } from 'drizzle-orm';

// Prepare a query with placeholder
const getUserById = db.select()
  .from(users)
  .where(eq(users.id, sql.placeholder('id')))
  .prepare('get_user_by_id');

// Execute
const user = await getUserById.execute({ id: 1 });
```

**Placeholder pattern:** Use `sql.placeholder('userId')` for parameterized queries. The prepared statement is compiled once and reused with different values.

**Performance:** Prepared statements enable query plan caching and reduce overhead on repeated executions — ideal for hot paths like session creation.

### Supabase Postgres Best Practices

- **Index usage:** Ensure `member_userId_idx` exists for fast lookups on `member.userId`. Current schema has `index("member_userId_idx").on(table.userId)`.
- **Joins:** Use inner join (default in Drizzle) for member → organization since we only care about memberships that have a valid organization.
- **Ordering:** Use `member.createdAt ASC` with `limit(1)` to get the first membership.

## Implementation Reference

```typescript
// packages/db/src/queries/get-initial-organization.ts
import { eq, asc, sql } from "drizzle-orm";
import { db } from "../connection";
import { member, organization } from "../schema";

const getInitialOrganizationStmt = db
  .select({
    id: organization.id,
    slug: organization.slug,
  })
  .from(member)
  .innerJoin(organization, eq(member.organizationId, organization.id))
  .where(eq(member.userId, sql.placeholder("userId")))
  .orderBy(asc(member.createdAt))
  .limit(1)
  .prepare("get_initial_organization");

export async function getInitialOrganization(userId: string): Promise<{ id: string; slug: string } | null> {
  const rows = await getInitialOrganizationStmt.execute({ userId });
  const row = rows[0];
  return row ? { id: row.id, slug: row.slug } : null;
}
```
