## Description

Extend the auth layer to: (1) expose `organizationClient()` on the auth client for operator app usage; (2) add a `databaseHooks.session.create.before` hook that populates `activeOrganizationId` and `activeOrganizationSlug` on new sessions by calling `getInitialOrganization(session.userId)`; (3) add `activeOrganizationSlug` to session `additionalFields` (and ensure the session table has the column); (4) run a migration if the session table lacks `activeOrganizationSlug`.

**Step-by-step implementation:**

1. **Auth client:** In `packages/auth/src/client.ts`, add `organizationClient()` from `better-auth/client/plugins` to the plugins array. The client already has `adminClient()` and `oneTimeTokenClient()`.

2. **Session additionalFields:** In `packages/auth/src/index.ts`, add `activeOrganizationSlug` to `session.additionalFields` (type: `string`, required: false, input: true). The session already has `impersonatedBy` and `activeOrganizationId` in the schema; verify whether the organization plugin provides `activeOrganizationSlug` — if not, add it.

3. **Database migration:** The schema at `packages/db/src/schema/auth.ts` has `activeOrganizationId` but NOT `activeOrganizationSlug`. Add `activeOrganizationSlug` column to the session table and generate/run a migration.

4. **databaseHooks:** Add `databaseHooks` to the Better Auth config in `packages/auth/src/index.ts`. In `session.create.before`, call `getInitialOrganization(session.userId)` (import from `@slushomat/db`), then return `{ data: { ...session, activeOrganizationId: org?.id ?? null, activeOrganizationSlug: org?.slug ?? null } }` so the session is created with these fields.

5. **Hook placement:** The `databaseHooks` config lives at the top level of `betterAuth({ ... })`. The hook receives the session object; modify and return the updated data.

## Research / Context7

### Better Auth databaseHooks.session.create.before

From web search and Better Auth docs:

- **Hook signature:** `databaseHooks.session.create.before` receives the full session object when creating a new session.
- **Return value:** Return `{ data: { ...session, newField: value } }` to modify the session data before it is persisted.
- **additionalFields:** Any field added in the hook must be declared in `session.additionalFields` first (type, required, input).
- **Secondary storage:** If using Redis/secondary storage for sessions, ensure Better Auth version supports persisting hook-modified data (v1.1.3+).

## Implementation Reference

```typescript
// packages/auth/src/index.ts
import { getInitialOrganization } from "@slushomat/db";

export const auth = betterAuth({
  // ...existing config
  database: drizzleAdapter(db, { provider: "pg", schema }),
  session: {
    additionalFields: {
      impersonatedBy: { type: "string", required: false, input: true },
      activeOrganizationSlug: { type: "string", required: false, input: true },
      // Ensure activeOrganizationId is in additionalFields if not from org plugin
    },
  },
  databaseHooks: {
    session: {
      create: {
        before: async (session) => {
          const org = await getInitialOrganization(session.userId);
          return {
            data: {
              ...session,
              activeOrganizationId: org?.id ?? null,
              activeOrganizationSlug: org?.slug ?? null,
            },
          };
        },
      },
    },
  },
  plugins: [admin(), oneTimeToken(), organization({...})],
});
```

## Risk / Notes

- **Organization plugin schema:** The Better Auth organization plugin may add `activeOrganizationId` and `activeOrganizationSlug` to the session table automatically. Check the organization plugin migration/schema; if it already adds these columns, only add to `additionalFields` if needed for type/validation. The project's custom schema in `packages/db` may override — ensure the column exists in the actual DB.
- **Null handling:** When the user has no org, set both fields to `null`. The operator app will redirect to `/invitations` when `organization.list()` is empty.
- **Import path:** Use `import { getInitialOrganization } from "@slushomat/db"` — ensure T02 exports it from the package root or a documented path.
